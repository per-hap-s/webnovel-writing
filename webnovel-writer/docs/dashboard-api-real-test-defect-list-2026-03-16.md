## 后续验收口径更新

以下缺陷结论用于保留 2026-03-16 的历史基线。自第三阶段起，`plan` 冷启动的正式验收口径改为：

- bootstrap 后无需手工先改 `大纲/总纲.md`
- 首轮 `plan` 默认依赖 bootstrap 元数据、planning-profile 和总纲骨架形成最小输入闭环
- 若输入仍不足，任务必须以 `PLAN_INPUT_BLOCKED` 失败，并返回结构化 `blocking_items`

## 缺陷分级说明
- `P1`：阻断主验收目标，必须优先修复
- `P2`：不阻断当前两章闭环，但会阻断下一轮主链扩展或稳定性验收
- `P3`：不阻断系统闭环，主要影响内容清晰度、可维护性或后续连续性

## 缺陷列表

### DEFECT-001 `plan` 在真实 API 下仍不稳定，无法纳入主链通过项
- 优先级：`P1`
- 影响范围：`plan volume=1`
- 现象：
  - 初次 `plan` 对空的 `大纲/总纲.md` 非常敏感，无法有效产出
  - 手工补足总纲后，不再立即阻断，但仍会长时间停留在运行态
  - 本轮两章测试无法把“首次 `plan` 成功”作为通过项
- 复现条件：
  - 新建项目后直接执行 `plan`
  - 或在已有较弱总纲上下文下执行 `plan`
- 实际结果：
  - `plan` 无法稳定在可接受时间内完成并写回规划结果
- 预期结果：
  - `plan` 应能在真实 API 模式下稳定完成并产出可供后续 `write` 消费的规划文件
- 影响判断：
  - 当前阻断“完整主链验收”
  - 会影响后续把 `plan -> write -> review -> resume` 作为统一回归链路
- 建议修复方向：
  - 继续检查真实 API 模式下 `plan` 的超时预算、重试策略、解析容错和写回阶段
  - 明确区分“输入不足导致阻断”和“上游响应长时间无结果”两类失败
- 建议复测：
  - 新项目冷启动执行 `plan volume=1`
  - 弱总纲 / 强总纲 两种输入各复测一次

### DEFECT-002 `plan` 入口对项目初始大纲质量过于脆弱
- 优先级：`P2`
- 影响范围：首次项目初始化后的规划体验
- 现象：
  - 项目虽已 bootstrap 成功，但若 `大纲/总纲.md` 内容过少，`plan` 难以进入有效规划
  - 需要人工补写总纲后才能继续真实写作测试
- 复现条件：
  - 使用刚初始化的新项目直接执行 `plan`
- 实际结果：
  - 系统不能稳定给出可执行的首轮卷规划
- 预期结果：
  - 初始化后的默认项目状态应支持首轮 `plan` 起步，至少给出可用的阻断提示或半结构化最小规划
- 影响判断：
  - 不一定阻断写作引擎本身
  - 但会明显降低 Dashboard 首次可用性
- 建议修复方向：
  - 在 `plan` 前增加更明确的最小输入校验和引导
  - 或为 bootstrap 默认生成更完整的最小总纲骨架
- 建议复测：
  - 空目录初始化后不做人工编辑，直接执行 `plan`

### DEFECT-003 Chapter 2 章尾的“买药信息”表达存在事实摇摆感
- 优先级：`P2`
- 影响范围：Chapter 2 内容一致性与读者理解
- 对应章节：`D:\CodexProjects\NovelTest-2Ch\正文\第0002章.md`
- 现象：
  - 前文已明确妹妹要买的是退烧药
  - 章尾又写成“是退烧药吗？还是止咳的？他竟然记不清了”
  - 读者容易误读为前后事实冲突，而不是主角认知受损
- 实际结果：
  - review 已将其识别为中等严重度问题
- 预期结果：
  - 应明确呈现为“客观信息没变，主角读取和记忆能力发生错位”
- 影响判断：
  - 不阻断系统闭环
  - 会影响后续读者对规则代价的理解精度
- 建议修复方向：
  - 在章尾补一笔主观认知标记，明确这是记忆代价，不是设定改写
- 建议复测：
  - 重审 chapter 2 的 consistency review，确认该问题消失或降为 low

### DEFECT-004 Chapter 2 第二轮行动的时间锚点不够显式
- 优先级：`P3`
- 影响范围：Chapter 2 连续性
- 对应章节：`D:\CodexProjects\NovelTest-2Ch\正文\第0002章.md`
- 现象：
  - 本章明确出现 `22:17` 回档点和 `22:20` 观察点时间
  - 但中间一系列采购、改路、设局、反制、取证、接电话动作缺少额外时间锚点
- 实际结果：
  - 逻辑上仍能推导成立
  - 但需要读者自行补算时间窗口
- 预期结果：
  - 时间窗口应更容易被直接验证
- 影响判断：
  - 不阻断当前两章通过
  - 会增加后续多章连续时间线维护成本
- 建议修复方向：
  - 在第二轮行动中补一到两个短时间锚点
- 建议复测：
  - chapter 3 写作前后重新执行 continuity review，确认时间线风险降低

### DEFECT-005 Chapter 2 关键场景的空间关系说明偏弱
- 优先级：`P3`
- 影响范围：Chapter 2 连续性与临场感
- 对应章节：`D:\CodexProjects\NovelTest-2Ch\正文\第0002章.md`
- 现象：
  - 从便利店门口转入维修通道的路线能大致看懂
  - 但便利店、居民楼、死巷、维修通道之间的相对位置没有被明确立住
- 实际结果：
  - 不影响系统写回或索引
  - 影响读者对追逐和反制路线的空间理解
- 预期结果：
  - 关键反杀舞台应有最小必要的方位说明
- 影响判断：
  - 非系统缺陷
  - 属于内容层连续性瑕疵
- 建议修复方向：
  - 在转场句中补半句方位关系
- 建议复测：
  - chapter 2 再审 continuity review

### DEFECT-006 第 1 章预警信息与第 2 章新预警来源仍偏悬空
- 优先级：`P2`
- 影响范围：Chapter 1-2 跨章伏笔管理
- 对应章节：
  - `D:\CodexProjects\NovelTest-2Ch\正文\第0001章.md`
  - `D:\CodexProjects\NovelTest-2Ch\正文\第0002章.md`

  - 两类预警来源是否同源，目前仍缺最小辨识

  - 第 2 章完成了承接，但没有给出来源方向
- 预期结果：
  - 至少应给主角一个初步判断：同源、异源、或无法判断但有口吻差异
- 影响判断：
  - 不阻断当前两章闭环
  - 会影响第 3 章及后续伏笔健康度

- 建议复测：

### DEFECT-007 新组织和制度性设定已进入正文，但结构化设定仍未同步
- 优先级：`P2`
- 影响范围：后续长期连续性、设定管理
- 现象：
  - Chapter 2 已落下“雨幕中介”“调查局”“异常观察点临时巡查时间”等新设定
  - 但 review 仍提示结构化设定资料未同步
- 实际结果：
  - 当下写作未受阻
  - 后续章节若继续引用，易产生称呼、权限、职责漂移
- 预期结果：
  - 新组织和世界规则应及时进入项目结构化设定
- 影响判断：
  - 当前不阻断两章写作闭环
  - 会提升后续 3-5 章连续性风险
- 建议修复方向：
  - 检查 data-sync 或设定同步链路是否需要补充实体登记
- 建议复测：
  - 写完 chapter 3 后检查 state / 设定集 / 实体索引是否已更新
