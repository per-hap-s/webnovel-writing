Webnovel Writer 快速说明
========================

1. 默认启动方式
双击 `Start-Webnovel-Writer.bat` 后，会直接打开 Web Dashboard 工作台。
现在不会再弹启动前菜单，也不会再要求先选目录或先初始化项目。

2. 在工作台里做什么
工作台首页可以：
- 打开已有项目
- 新建项目
- 切换当前项目
- 在左侧项目轨道直接看到当前 / 固定 / 最近项目卡
- 清理失效记录
- 设置默认落地方式

3. 工具页
工作台工具页可以直接执行：
- 登录 Codex CLI
- 打开当前项目终端
- 打开快速说明
- 开启局域网访问

4. 新建项目
点击“新建项目”后：
- 先选一个目录
- 再填写标题和题材
- 创建完成后，规划信息默认留空，等你自己确认后再保存
- 创建成功后会自动进入该项目，并带上 planning 提示

5. 打开已有项目
点击“打开已有项目”后：
- 先选目录
- 如果目录已经初始化，会直接进入项目
- 如果目录还没初始化，页面会提示你改为新建项目

6. 配置 API
进入项目后，推荐在：
`总览 -> API 接入设置`

填写：
- 写作模型 Provider / Base URL / 模型名 / API Key
- RAG Base URL / Embedding 模型 / Rerank 模型 / API Key

保存后会写入项目根目录 `.env`。

7. 推荐顺序
1. 打开工作台
2. 打开已有项目或新建项目
3. 在项目页确认 Planning Profile 和 API 设置
4. 再运行 `plan / write / review / repair`

8. 兼容入口
- `tools\Start-Webnovel-Writer.bat menu`
- `tools\Start-Webnovel-Writer.bat dashboard-lan`
- `tools\Start-Webnovel-Writer.bat login`
- `tools\Start-Webnovel-Writer.bat shell`
- `.\Launch-Webnovel-Dashboard.ps1`

说明
----

- `.\Launch-Webnovel-Dashboard.ps1` 不传 `-ProjectRoot` 时，会直接启动全局工作台。
- 只有显式传入一个已初始化项目目录时，才会直接进入单项目模式。
- 如果界面和旧说明不一致，以当前 Dashboard 界面为准。
