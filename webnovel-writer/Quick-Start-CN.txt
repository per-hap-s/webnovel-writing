Webnovel Writer 启动说明
========================

1. 启动 Dashboard
在项目目录双击 `Start-Webnovel-Writer.bat`。
默认建议先选菜单 `1`，在本机启动 Dashboard。
如果只是自己在这台电脑上使用，这是最稳妥的入口。

2. 局域网访问 Dashboard
如果需要手机或局域网其他设备访问，选菜单 `2`。
启动后窗口会显示一个类似 `http://192.168.x.x:8765` 的地址。
手机和电脑必须在同一局域网，且 Windows 防火墙需要放行该端口。

3. 登录 Codex CLI
如果你准备用 Codex CLI 作为写作执行器，先选菜单 `3` 完成登录。
当未配置写作模型 API、但本机存在可用 Codex CLI 时，Dashboard 会自动回退到 CLI 模式。

4. 首次项目初始化
如果你选择的目录还不是小说项目，启动器会自动询问：
- 小说标题
- 题材

随后会在该目录创建 `.webnovel/state.json` 和基础目录结构。

5. 面板里修改 API
启动 Dashboard 后，进入：

`总览 -> API 接入设置`

你可以直接修改两组配置：

- 写作模型 API
  需要填写 `Provider`、`Base URL`、`模型名称`、`API Key`
- RAG API
  需要填写 `Base URL`、`Embedding 模型`、`Rerank 模型`、`API Key`

保存后会发生三件事：

- 写入项目根目录 `.env`
- 立即刷新当前面板状态
- 后续任务直接使用新配置

如果 API Key 输入框留空，则保留当前已保存的 Key，不会被清空。

6. 面板并非纯只读
当前面板除了查看数据，还支持：

- 创建项目
- 发起任务
- 重试任务
- 批准或拒绝回写
- 确认失效事实
- 保存 API 配置

文件浏览页仍然是只读，并且只允许读取 `正文`、`大纲`、`设定集`。

7. 推荐使用顺序

1. 先选菜单 `3` 登录 Codex CLI，或在面板里填好写作模型 API
2. 再选菜单 `1` 启动 Dashboard
3. 在 `总览` 页确认项目状态和 API 状态
4. 从 `任务入口` 发起 `init / plan / write / review / resume`

8. 备用入口

- 本机面板：`.\Launch-Webnovel-Dashboard.ps1`
- CLI 面板：`webnovel dashboard`
- 打开项目终端：启动器菜单 `4`

说明
--------

- 推荐把 API 配置保存在每个项目自己的 `.env` 中，避免多本书串配置。
- 如果说明和界面不一致，以 Dashboard 页面里的 `API 接入设置` 和状态提示为准。
